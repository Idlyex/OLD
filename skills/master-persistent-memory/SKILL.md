Master Persistent Memory
Overview
This skill establishes a true, robust, and self-improving persistent memory system that transforms the AI from a stateless responder into a continuously evolving, highly intelligent, long-term agent. It enforces strict memory retrieval and updating protocols on every single interaction, enabling deep context retention, progressive learning from experience, prevention of repeated mistakes, and exponential growth in capability and alignment with the user over time.

When to Use This Skill
This skill is foundational and mandatory. It must be active in every single interaction without any exception, especially when:

Working on long-term projects or codebases
Maintaining continuity across multiple sessions
Learning from past decisions and mistakes
Personalizing responses to user style and preferences
Building cumulative knowledge about architecture and performance
Evolving strategic understanding of the project
Preventing regression and repeated errors
Creating a true "evolving agent" experience
Core Capabilities

1. Mandatory Memory Retrieval Protocol
At the very beginning of every interaction, the AI must systematically load and thoroughly internalize all relevant memory files before any reasoning begins.

Apply when:

Starting any new session or response
Switching between different topics or tasks
Continuing work on ongoing projects
Retrieval framework:

Always load core_memory.md first as the single source of truth
Load memory_index.md to understand memory landscape
Load all contextually relevant specialized memory files
Cross-reference past decisions, mistakes, and user preferences
Maintain full awareness of project evolution and current state
2. Intelligent Memory Updating & Evolution
After every meaningful interaction, the AI must actively update and refine the memory system with new high-quality information.

Apply when:

Completing any significant task
Making important decisions
Discovering new insights or mistakes
Receiving user feedback on style or preferences
Achieving performance improvements or optimizations
Update framework:

Record new architectural decisions and rationale
Document mistakes and lessons learned in detail
Capture user preferences, dislikes, and working style
Save measurable results (benchmarks, before/after metrics)
Update project roadmap and current priorities
Note what works particularly well with this specific user
3. Structured Knowledge Retention
Maintain highly organized, searchable, and accurate records across multiple specialized domains.

Key Memory Areas:

Project architecture and design decisions
Codebase structure, critical modules, and dependencies
User coding style, preferences, and anti-patterns
Past mistakes, their causes, and prevention strategies
Performance benchmarks and optimization history
Profiling results and identified bottlenecks
Strategic decisions and their outcomes over time
Evolution of the agent's own capabilities
4. Self-Evolution Mechanism
Actively analyze historical interactions to improve future behavior and recommendations.

Apply when:

Identifying patterns in user feedback
Noticing repeated types of mistakes
Refining understanding of user priorities
Improving recommendation quality over time
Evolution framework:

Detect patterns in what the user likes and dislikes
Refine recommendations based on past successes and failures
Gradually align more closely with user's thinking style
Improve the quality and relevance of proactive suggestions
5. Smart Retrieval & Relevance Engine
Dynamically determine which memory files and specific sections are most relevant to the current task and load them efficiently.

6. Memory Consolidation & Maintenance
Periodically review and consolidate the memory system to prevent bloat while preserving important knowledge.

Apply when:

Memory grows significantly
Old information becomes outdated
Higher-level insights can be synthesized
Consolidation process:

Summarize old detailed entries into higher-level insights
Archive rarely used information
Remove truly obsolete data
Create new generalized principles from specific cases
7. Memory Quality Assurance
Maintain extremely high standards for memory accuracy, clarity, and usefulness.

Quality standards:

All entries must be dated and contextualized
Distinguish clearly between facts, observations, and interpretations
Include measurable results whenever possible
Write entries that will remain useful months later
Maintain consistency in structure and terminology

Application Guidelines

Strict Operational Protocol (must be followed every time):

Start of Interaction

Immediately load core_memory.md and memory_index.md
Load all additional relevant memory files
Fully internalize context before any reasoning
During Reasoning

Actively reference past decisions, mistakes, user preferences, and lessons
Avoid repeating previously identified bad patterns
Leverage historical knowledge for better recommendations
End of Interaction

Perform comprehensive memory updates
Update core_memory.md with the most important changes
Update specialized files as needed
Ensure all new insights are properly recorded
Memory Update Rules:

Be concise but extremely information-dense
Always include dates and context
Clearly separate facts from observations
Record measurable results whenever possible
Note user reactions and explicit preferences
Write entries that will be valuable months later
Remember

Persistent Memory is about:

Creating true continuity across sessions
Preventing repeated mistakes
Building deep understanding of the user and project
Enabling exponential agent improvement over time
Transforming the AI into a genuine long-term partner
Always distinguish between:

Short-term memory (current conversation)
Long-term memory (persistent knowledge)
Facts and observations
Temporary decisions and permanent lessons
Goals of Persistent Memory:

Never forget critical context
Never repeat the same mistakes
Continuously become more aligned with the user
Accumulate valuable project and performance knowledge
Enable increasingly higher-quality proactive suggestions
